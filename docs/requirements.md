# 需求文档 — 手势互动彩色画面

## 1. 项目概述

通过摄像头获取实时画面，网站初始化显示黑白画面。用户通过手势与画面交互。

目标平台：Web，支持手机端与电脑端。

## 2. 核心功能

### 2.1 摄像头显示
- 使用 getUserMedia 获取摄像头画面
- 默认黑白滤镜显示
- 支持响应式布局，适配手机与电脑端

### 2.2 双手拍照手势 → 局部原彩
- 双手形成矩形框，手掌摊开
- 手势矩形范围内画面恢复原始色彩，其余区域保持黑白
- 原彩仅在手势出现时触发，手势消失恢复全黑白
- 矩形边缘圆角 + 细边框，视觉自然

### 2.3 单手捏手势 → 手绘轨迹
- 单手大拇指和食指捏紧触发手绘（其余三指须伸展，防止误触）
- 粉色轨迹（Catmull-Rom 平滑曲线）
- 轨迹粗细根据手速自动调整（2px–12px）
- 支持保存手绘内容（截图导出）

### 2.4 手势防误触机制
- 迟滞阈值：进入/退出使用不同灵敏度
- 5 帧确认进入 + 8 帧确认退出
- 捏手势验证：z 轴深度（两指尖同平面）、食指中间关节伸展、其余三指伸展
- 捏手势保持 10 帧后才开始绘制（draw arm），松手后 15 帧冷却期
- 拍照手势增加稳定性检查（矩形中心不跳动太快）
- EMA 关键点平滑（α=0.4）

## 3. 用户交互
- 页面半透明提示："双手拍照恢复原彩"、"单手捏手绘粉色轨迹"
- 8 秒自动隐藏提示
- 底部工具栏：撤销 / 清空 / 保存截图

## 4. 扩展功能（可选）
- 多人互动：多个手势同时触发不同彩色区域
- Undo/Redo：撤销/重做手绘操作
- 手势自定义：用户可定义触发动作
- 导出功能：截图或录屏保存
